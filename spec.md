# オフィス座席表ツール（ch-seats-tool） - 仕様書

## 概要

- 目的: オフィスの「席替え」「レイアウト編集」「座席表の閲覧」を 1 つで担うツール。
  座席表の更新作業を楽にし、複数拠点のレイアウト管理を可能にする。
- 対象ユーザー:
  - 編集者（総務・席替え担当者など）: レイアウト作成、メンバー割り当て、データ更新
  - 閲覧者（社内メンバー全般）: 座席表の閲覧、人・席の検索
- 配布先: zaproom（index.html をトップレベルに含む zip をアップロード）
- 規模感: 数拠点、1 拠点あたり〜100 席程度

## アーキテクチャ方針（最重要）

**データとロジックを分離する。** ツール本体（zip）はロジックのみを持ち、座席データはすべて zaproom の **Session Datastore**（サーバー側共有ストア）に保存する。

- 閲覧者・編集者とも同じセッション URL を開くだけで常に最新の座席表が見える
- 編集はアプリ内の操作だけで完結し、保存すると即座に全員へ共有される。**zip の再アップロードは機能追加時のみ**（特定の管理者の手を介さず誰でも更新できる、という要件を満たす）
- API 仕様は `docs/zaproom-session-datastore.md` を参照（GET/PUT /api/sessions/{sessionId}、ETag 楽観ロック、PUT 時の x-amz-content-sha256 ヘッダー必須、credentials: 'include' 必須）

### manifest.json（zip トップレベル）

```json
{
  "version": 1,
  "features": {
    "session_datastore": {
      "enabled": true,
      "visibility": ["internal", "private"],
      "write_scope": ["internal", "author-only"]
    }
  }
}
```

- 通常運用: visibility = internal / write_scope = internal のセッションを 1 つ作り、その URL を社内に共有（全員閲覧可・全員編集可）
- 編集権限の制御は設けない方針（誰でも編集できることを優先。アプリ内の権限管理は実装しない）
- private セッションは席替え案の下書きなどに利用可能

### データ保存の挙動

- 読み込み: 起動時に GET。以降は 30 秒間隔のポーリング + 手動更新ボタンで最新化（閲覧者向け）
- 保存: 編集操作後に明示的な「保存」ボタンで PUT（編集途中の状態を撒き散らさないため自動保存はしない。未保存変更はインジケータ表示）
- 競合: 409 時は最新データを取得し、ユーザーに通知してマージ or 再適用を促す（saveWithRetry パターン）
- 図面画像は dataURL として session データ内に保持。アップロード時にクライアント側で自動縮小・圧縮（長辺 2000px・JPEG/WebP、1 拠点 500KB 目安）
  - **リスク**: Session Datastore のサイズ上限は未確認。フィードバックフェーズで実データ量（数 MB 想定）の保存を検証すること
- sessionId が無い（セッション未選択）場合は案内画面を表示

## 技術構成

- ビルドあり構成: React + Tailwind CSS + shadcn/ui + Vite 8（node v22 利用可能）
- TypeScript 化済み（src/ は .ts/.tsx、strict モード。`npm run typecheck` で検証。ドメイン型は lib/model.ts に定義）
- `base: './'` を vite.config.js に指定（zaproom 配信要件）
- ローカル開発: vite-plugin-session-datastore.mjs（Session Datastore エミュレータ）を使用
- zip: dist/ の中身をトップレベルに（index.html, manifest.json, spec.md, favicon 等）

## UX 方針（フィードバック反映 2026-06-11）

**「名前直接入力ファースト」**: メンバー登録なしで座席表が完成する。実物の座席表 PDF（CAD 図面 + 名前ラベル）を最短手順で再現できることを最優先とする。

- 図面をアップ → クリックで席を置く → その場で名前をタイプ → Enter で確定し連続配置、だけで完成
- メンバー台帳（あだ名・アイコン・Slack・プロフィール自己編集）は**オプション機能**。席の名前と台帳の紐付けは後からでもできる
- 閲覧はセッション URL を開くだけ（インポート等の操作は一切不要。データ作成は最初の 1 人がやれば全員に共有される）

### 変更履歴（2026-06-11 UX 改善の実装反映）

「座席表を作るのがめんどくさい」への対策として、仕様策定後に以下を追加実装した。

- **インライン名前入力の強化**: Enter 確定（IME 対応）/ Esc キャンセルに加え、**Tab / Shift+Tab で確定して読書順（同じ行の隣 → 次の行の左端）の席へ入力を連続移動**
- **一列スタンプ配置**: 配置モード中のドラッグで席を等間隔に一列プレビュー配置（水平/垂直 ±10° は軸スナップ）。Undo 1 回で取り消し可
- **スマートスナップ**: 席の移動・配置時に他席の X/Y へ吸着しガイド線を表示（Alt で無効化）。無地キャンバスは 50px グリッドにも吸着
- **複数選択と一括操作**: Shift+クリックでトグル選択 / Shift+ドラッグで矩形選択。一括移動・左揃え/上揃え・横/縦等間隔・一括 90° 回転（R キー）・島ごと複製・一括削除
- **Undo/Redo を必須機能に昇格**: Cmd/Ctrl+Z / Shift+Cmd/Ctrl+Z（履歴 50 件、ドラッグ移動は 1 ステップに集約）。Delete/Backspace で選択席削除
- **名前リスト流し込みの拡張**: 改行区切りに加えスプレッドシートの複数列貼り付け（名前列・部署列を自動推定 → 選択可）。部署列があれば「台帳にも登録して部署色分けを有効化」を選べる。空席への読書順一括割り当て / スキップ / 1 つ戻す（Cmd/Ctrl+Z）/ Esc 終了
- **オンボーディング**: 席ゼロの拠点では「① 図面アップロード ② クリックで席配置 / 名簿一括入力」のガイドカードを表示。図面設定直後・席ゼロで編集開始時は自動で連続クリック配置モードへ。拠点ゼロなら「編集をはじめる」で「フロア 1」を自動作成
- **図面まわり**: キャンバスへの PDF/画像のドラッグ&ドロップ取り込みに対応。図面差し替えで縦横比が変わる場合は既存座席を contain フィットで自動再マッピング。図面削除（無地キャンバスへ戻す。座席は保持）
- **席サイズ調整**: 拠点ごとに全席の表示倍率を 0.4〜2.0（10% 刻み）で調整可能（`Location.seatScale`、後方互換: 旧データは 1 扱い）
- **エリア（ゾーン）機能**: 図面上の色付き区画（部門エリア等）を矩形 + ラベル + プリセットカラーで再現（`AppData.zones`、後方互換: 旧データは [] 扱い）。詳細は「機能一覧」参照
- **席ごとのサイズ・向き（2026-06-11 第2ラウンド）**: 席単位のサイズ倍率 `Seat.w` / `Seat.h`（0.5〜6.0、ハンドルは 0.25 刻み。後方互換: 欠損時は 1）。単一選択時に右端/下端/右下角のリサイズハンドルと上部の回転ハンドル（15° スナップ、Shift でフリー回転）を表示。SeatToolbar にサイズプリセット（1×1 / 2×1 / 3×1）。既存の R キー・±90° ボタンは継続
- **矩形選択の Shift 不要化（2026-06-11 第2ラウンド）**: 編集モードの select ツールでは背景の左ドラッグ＝矩形選択（Shift+ドラッグは既存選択への追加）。パンは Space+ドラッグ / 中ボタンドラッグ / 2本指スクロール / ピンチ。閲覧モードは従来どおり背景ドラッグ＝パン
- **コピー & ペースト（2026-06-11 第2ラウンド）**: Cmd/Ctrl+C で選択席（複数可）をアプリ内クリップボードへ、Cmd/Ctrl+V で貼り付け。マウスカーソルがキャンバス上にあればその位置を基準に相対配置を保持して貼り付け（なければ元位置から席1個分オフセット、連続ペーストはさらにオフセット）。名前・台帳紐付けは引き継がない（w/h/rotation/type/label は引き継ぐ）。貼り付け後は新規席群を選択。入力欄フォーカス中は発動しない。Undo 1 回で取り消し可
- **机テンプレート（2026-06-11 第2ラウンド）**: 編集モードの「テンプレート」メニューから「1人席 / 2人掛け長机（横並び）/ 4人島（2×2 対面）/ 6人島（3×2 対面）/ 会議テーブル（横長 meeting 席）」を選ぶとスタンプモードに。マウス追従の半透明プレビュー付きでクリックのたびに一式配置（対面島は向かい合う 180° 回転、Esc で終了）。1 配置＝1 mutate＝Undo 1 回
- **席クリックの操作モデル変更（2026-06-12）**: 編集モードの席シングルクリックは「選択のみ」に変更（直後の Delete/Backspace で削除可能）。インライン名前編集は**席のダブルクリック**または**単一選択中の Enter** で開く（配置モードの新規席自動入力・Tab 連続入力・流し込みは従来どおり）。席とエリアの選択は排他で、席の上の操作はエリアに届かない
- **その他**: 未保存変更があるままのタブ閉じ/リロードで確認ダイアログ。検索ヒット時は非該当席をディム表示。台帳紐付き席に別名を直接入力すると紐付けを解除して上書き / 紐付け解除時は表示名を直書き名に引き継ぐ。席の直接入力名をワンクリックで台帳へ新規登録して紐付け可能。メンバー台帳パネルはデフォルト閉（右端の縦タブから開く）
- **複数選択の 90° 回転をグループ回転に変更（2026-06-12）**: 個別に席が回るのではなく、選択全体を 1 つの塊として bbox 中心の周りで回転（位置も回る。図面の縦横比による歪みはピクセル空間で回転して回避）
- **閲覧モードの表示名トグル（2026-06-12）**: ヘッダーの「表示: あだ名 / 本名」ボタンで座席の名前表示を切り替え（台帳紐付け席のみ対象。閲覧者ごとの好みなので localStorage に保存しセッションデータには含めない）
- **等間隔の修正とクラスタ化（2026-06-12）**: 横/縦等間隔を「同じ列/行の席はまとめて動かし、席サイズ・回転込みの余白が均等になる」配分に修正（従来は全席の中心を均等割りしており、島選択で列が崩れて席が重なっていた）。収まらない場合は最小余白を確保して先頭から並べ直す
- **事業部リストの設定化（2026-06-12）**: 事業部マスタをコード埋め込みからセッションデータ（`AppData.settings.divisions`）へ移動。編集モードの歯車（設定）から行の追加・コード/名称編集・削除・並べ替えが可能。zip は社内の誰でもダウンロードできるため組織固有リストをロジック側に持たない方針 + 組織変更時に zip 再アップ不要。使用中の事業部を削除した場合、メンバーのコードは保持し名称解決できないときはコードを表示
- **社員名簿との差分同期（2026-06-12）**: メンバー台帳パネルの「名簿から同期」で名簿スプレッドシートを貼り付け → 列マッピング（メールアドレス必須=突き合わせキー）→ 差分プレビュー（新規 / 更新(変更点表示) / 名簿にいない）→「名簿にいない」メンバーは確認ステップで「退職にする / そのまま残す」を選択（勝手に削除しない）→ 1 mutate で一括反映（Undo 1 回）。同期で更新するのは名簿由来フィールド（本名・あだ名・事業部・部署・Slack・status）のみで、アイコン・メモ・席の紐付けは保持。名簿が正・台帳はキャッシュという位置付けで二重管理を解消。退職メンバーは台帳一覧からデフォルト非表示（トグルで表示）、席に紐付いたままの場合は席に警告スタイル + ポップオーバーに「退職」バッジ
- **メンバーの事業部 (division) フィールド追加（2026-06-12）**: 従来の「所属 (department)」UI を「事業部」(固定リストのプルダウン・コード保存/名称表示・未設定可) と「部署」(課・チームなどの自由入力。旧 department をそのまま継続=後方互換) に分離。事業部マスタは `lib/model.ts` の `DIVISIONS`（23 件）。色分けキーは「事業部 (設定時) → 部署 (フォールバック)」(`memberColorKey`)。検索は事業部の名称・コード・部署のいずれでもヒット。CSV 取り込みに「事業部」列マッピングを追加（コード完全一致 → 名称完全一致 → 名称部分一致が一意のときのみ解決。解決不可なら空のまま）。MemberEditDialog / ProfileEditDialog に事業部 Select + 部署入力、MemberPanel / SeatDetailPopover / MemberLinkDialog は「事業部名称 / 部署」を併記表示
- **机間の余白調整（2026-06-12）**: 複数選択ツールバーに「間隔 − / ＋」を追加。選択全体の中心から各席までの距離を 10% ずつ拡縮（連打は Undo 1 ステップに集約）
- **改修 2026-06-12 (v2)**: 事業部デフォルトリスト削除 / MemberPanel ボタン統合 / メール必須化 / ラベル簡潔化
  - `DEFAULT_DIVISIONS`(特定組織固有のリスト)を削除。`createDefaultSettings()` の divisions は空配列に変更。旧データで settings が無い場合も空配列で補完(組織固有情報はコードに持たず、すべて設定から投入する)
  - MemberPanel のボタンを「追加」「名簿から取り込み / 同期」の2つに統合。CsvImportDialog は削除し、その機能は RosterSyncDialog に吸収（email列なし→一括追加モード、email列あり→差分同期）
  - MemberEditDialog でメールアドレスを必須化。メール空のメンバーは台帳パネルで警告アイコン表示
  - フォームラベルの「(補足)」形式を削除し簡潔化

## 機能一覧

### 必須機能

- [ ] 拠点（フロア）管理: 拠点の追加・名称変更・削除・並べ替え。拠点タブ/セレクタで切替
- [ ] 図面アップロード: 拠点ごとにフロア図面（PNG/JPG/**PDF**）を背景に設定。PDF はクライアント側で画像化（pdf.js）。自動縮小・圧縮
- [ ] 座席レイアウト編集: 図面上にクリック（またはダブルクリック）で座席を配置 → **その場でインライン名前入力 → Enter で確定**。ドラッグ移動・複製・削除・回転（ハンドル/R キー）・席ごとのサイズ変更（リサイズハンドル/プリセット）・Cmd/Ctrl+C/V のコピペ・机テンプレート一式配置。図面なしでも無地キャンバス+グリッドで配置可能（机が固定でない拠点向け）
- [ ] 座席への名前直接入力: 座席は自由テキストの名前を持つ（メンバー登録不要）。席クリック → 名前編集。空席はラベルなしの枠
- [ ] 名前リスト流し込み: 名簿やリストを貼り付け → 席を順にクリックして次々に割り当てる高速入力モード
- [ ] 座席の入れ替え（スワップ）・移動: ドラッグ操作。席替えは手動操作のみ（シャッフル機能は不要）
- [ ] エリア（ゾーン）: 「営業エリア」「フレキシブルエリア」のような色付き区画を図面上に再現。編集モードの「エリアを追加」ツールでドラッグして矩形を描く → その場でラベルをインライン入力 → プリセットカラー（12 色・半透明）から選択。既存エリアはクリックで選択し、ドラッグ移動・四隅ハンドルでリサイズ・ラベル/色変更・削除（Delete キー可）。座席より背面（図面 → ゾーン → 座席の z 順）に半透明塗り + 左上ラベルで描画し、閲覧モードでも表示（操作不可）。Undo/Redo 対応（移動・リサイズは 1 ステップに集約）

### オプション機能: メンバー台帳（後から使いたい人向け）

- [ ] メンバー管理: 本名・あだ名・所属（部署/チーム）・アイコン画像（自動縮小、任意）・Slack 情報(任意)の登録・編集・削除
  - 社員名簿（スプレッドシート）からの貼り付け/CSV インポート（「貼り付け → 列マッピング指定 → プレビュー → 取り込み」の汎用 UI）
  - あだ名・アイコン・Slack 情報などの細かいプロフィールは**本人が各自で設定**できる（全員書き込み可なので認証は不要）
  - 席の自由テキスト名と台帳メンバーの紐付け（紐付けると詳細ポップオーバー・色分け・アイコン表示が有効になる）
  - 台帳メンバーを席へドラッグ&ドロップで割り当てることも可能
### 必須機能（続き・閲覧/共通）

- [ ] 閲覧モード（初期表示）: 編集 UI を隠した表示。座席に名前を表示。台帳と紐付いた席はクリックで詳細（本名・所属・アイコン・Slack DM リンク）
- [ ] 検索: 名前（席の直書き名も対象）・部署で検索し、該当座席をハイライト
- [ ] 部署/チームごとの色分け表示（台帳と紐付いた席のみ）
- [ ] 座席種別: 固定席 / フリーアドレス / 会議席などの区別（色・スタイルで表現）
- [ ] Session Datastore 連携: 上記データ保存の挙動の通り
- [ ] JSON エクスポート/インポート: バックアップ・移行用の保険機能

### オプション機能（余裕があれば）

- [ ] 座席表の PNG 書き出し・印刷用表示（未実装）

## 画面構成

SPA 1 ページ + モード切替（閲覧 / 編集）。

### 画面1: 座席表ビュー（閲覧モード・初期表示）

- 拠点タブで切替し、図面上の座席と着席者を表示。検索ボックスでハイライト。部署色分け
- 座席/メンバー詳細ポップオーバーから「プロフィールを編集」（あだ名・アイコン・Slack 情報を本人がその場で更新して保存）
- 主要コンポーネント: LocationTabs, FloorMap, SeatNode, SearchBox, SeatDetailPopover, ProfileEditDialog

### 画面2: 編集モード（レイアウト+割り当て）

- 「編集」ボタンで切替。図面アップロード、座席の配置・移動・回転・削除、メンバー割り当て・スワップ
- サイドパネルにメンバー一覧（割り当て済/未割り当て）と CSV 一括登録
- ヘッダーに保存ボタン+未保存インジケータ
- 主要コンポーネント: FloorMap(編集), SeatToolbar, ImageUploader, MemberPanel, CsvImportDialog, SaveButton

### 画面3: データ管理ダイアログ

- JSON エクスポート/インポート、セッションの説明・運用ガイド表示
- 主要コンポーネント: DataIODialog

## データ構造（session データ）

```
AppData {
  version: number,
  locations: Location[],
  members: Member[],
  seats: Seat[],
  zones: Zone[],   // エリア (ゾーン)。2026-06-11 追加。欠損時は [] として正規化 (後方互換)
  settings: AppSettings,  // 2026-06-12 追加。欠損時はデフォルトで正規化 (後方互換)
  updatedAt: string (ISO8601)
```

## UI/UX 方針

- シンプル・実用重視。閲覧モードは誰でも迷わず使えるミニマル UI、編集機能はモード切替で隠す
- 図面の上に座席を重ねる視認性の高い表示（ズーム/パン対応）
- 操作は基本ドラッグ&ドロップ。タッチ操作（タブレット）も考慮
- 日本語 UI

## 運用フロー

1. 初回のみ: zip を zaproom にアップロード → ツール画面上部の SessionSelector で「組織内で共有 (internal) / 全員書き込み可 (internal)」のセッションを作成 → セッション付き URL を社内に共有
2. 日常: 閲覧者は URL を開くだけ。編集者はアプリ内で編集して保存ボタンを押すだけ（再アップロード不要）
3. 機能追加時のみ: zip を再ビルドして zaproom に再アップロード（データはセッション側にあるため失われない）
